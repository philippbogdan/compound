================================================================================
1. WHAT THIS PROJECT IS
================================================================================

`tribeux-server`           FastAPI orchestrator. Single thing the frontend talks
                           to. Runs the full pipeline. Port 8000.
`tribeux/app`              Vite/React frontend. Port 5174 (5173 was in use).
`tribeux-domtree`          DOM-extraction package, installed editable into the
                           server venv. No changes this session.
`urltovideo`               Local Python package the user wrote. URL → 256x256
                           30fps mp4 + page text. Has its own .git and venv.
                           Used by the new `render.py`.
TRIBE inference service    A SEPARATE FastAPI app, exposed via ngrok by the
                           user. Endpoints: GET /health, POST /score (multipart:
                           video, label). Currently at:
                             https://refined-sixties-elliptic.ngrok-free.dev
                           The URL is ephemeral — it changes between sessions.

Pipeline stages (orchestrator):
  render  → tribedomtree (DOM units + screenshot)
  encode  → urltovideo (mp4 + text + 13 sample frames)        ← NEW PATH
  tribe   → POST mp4 to <TRIBE_INFERENCE_URL>/score           ← LIVE
  claude  → Anthropic Sonnet 4.6 (real API)                   ← LIVE
  compose → tribedomtree.patch + v2 stub-shift inference
  done    → assemble Report


================================================================================
2. WHAT'S RUNNING RIGHT NOW (or how to start it)
================================================================================

Server (background):
  cd tribeux-server
  source .venv/bin/activate
  uvicorn tribeux_server.main:app --host 127.0.0.1 --port 8000 --reload
  # .env is auto-loaded from tribeux-server/.env then repo-root /.env

Frontend:
  cd tribeux/app
  npm run dev
  # served at http://localhost:5174 (or :5173 if free)
  # tribeux/app/.env has TRIBEUX_API_URL=http://127.0.0.1:8000 and
  # TRIBE_INFERENCE_URL=<ngrok URL> for reference

Health check:
  curl http://127.0.0.1:8000/api/health   # {"status":"ok"}

[3] TRIBE /score 404 GLITCH
    The first end-to-end live job got a fast 404 from the inference service
    on a perfectly-formed multipart POST; a manual retry the same way took
    the expected ~90s and returned a valid InferenceResult. Could be ngrok
    cold-start / free-tier flakiness, could be us. If it recurs:
      - the orchestrator already falls back to the stub on any exception
        (silently — only stderr); see inference.py `_post_score` callsite
      - log the raw response body (status + first 500B) before raising, so
        future failures don't disappear into the stub

[4] V2 INFERENCE IS STILL THE STUB-SHIFTER
    run_v2_inference doesn't call /score; it deterministically nudges v1's
    cohort_z + time-series by Claude's expected_delta. Per the README this
    is intentional ("v2 is a predicted delta"), but if the goal is to send
    the patched DOM screenshot back through TRIBE for an *actual* second
    inference, that's a separate task. Touchpoints:
      - pipeline.py compose stage builds full_v2 (bytes) when
        use_real_render and a patch lands; that screenshot would need to
        be wrapped in a 1-frame mp4 (or ideally a re-rendered scroll)
        before posting to /score.

[5] LEAKED ANTHROPIC KEY
    /.env contains a real ANTHROPIC_API_KEY that has been visible in this
    chat transcript. The README is explicit: "treat any key shared in
    chat or in a public diff as compromised." Rotate at:
      https://console.anthropic.com/settings/keys
    File is gitignored so the key won't end up in the commit, but the
    transcript itself is the leak vector.

[6] FRONTEND NEVER PASSES use_real_render
    Worked around by flipping the server-side default to True. If you ever
    need an offline/CI run via the UI, you'll need either a query param,
    a button, or the existing ?mock=1 mode (mockApi.js). Server can still

[7] urltovideo HAS NO PACKAGING
    `tribeux_server/render.py` adds the directory to sys.path. If urltovideo
    grows a pyproject, switch to `pip install -e ../urltovideo` and drop the

[8] NO TESTS ON DISK
    The two passing tests in section 4 only exist as inline shell scripts
    from this session. Worth committing them under tribeux-server/tests/.

[9] PERFORMANCE
    Each analyze job currently does Playwright TWICE: once inside urltovideo
    for the scroll recording, once inside render._full_screenshot for the
    full-page PNG. The second pass exists because record_scroll's video is
    cropped to the viewport and we want the full-page image for the report.
    If you care about speed, take the full-page screenshot from the page
    object inside record_scroll (would require modifying urltovideo) — saves
    ~10s per request.

8. QUICK REFERENCE — TRIBE /score CONTRACT
================================================================================

  POST <TRIBE_INFERENCE_URL>/score
  multipart/form-data:
    video  (required)  the mp4, content-type video/mp4
    label  (optional)  string identifier
    text   DO NOT SEND — user explicitly corrected this
  headers:
    ngrok-skip-browser-warning: true
  timeout: 300s (typical call ≈ 90s)
  response: same shape as samples/site_1.json / InferenceResult

If the orchestrator gets a fast 404 but a manual curl with the same payload
works, the diff is in how multipart is being built — not the model.
